Logfile::Rotate von Paul Gampe

Logrotation funktioniert häufig nach folgendem Schema:

  1. Verschiebe alte Logdateien bzw. die Sequenz der Logdateien
  2. Sende dem Prozess ein Signal, damit er die aktuell verwendete Logdatei schließt, und das Loggen kurzfristig unterbricht oder puffert
  3. Kopiere die aktuelle Logdatei in eine neue Datei, z.B. logfile.1
  4. Lösche oder trunkiere die aktuelle Logdatei.
  5. Sende dem Prozess ein Signal, damit er das Loggen wieder aufnimmt
  6. Räume auf, z.B. Dateien komprimieren oder uralte Dateien löschen

Eine kleine Skizze zur Verdeutlichung: 

 .-.       .-.
 | |  =>   | |
 `-'       `-'
logfile logfile.1

 .-.       .-.       .-.
 | |  =>   | |  =>   | |
 `-'       `-'       `-'
logfile logfile.1 logfile.2

usw.

 .-.       .-.       .-.       .-.       .-.       .-.         .--.
 | |  =>   | |   =>  | |   =>  | |  =>   | |  =>   | |    =>   ||||
 `-'       `-'       `-'       `-'       `-'       `-'         `--'
logfile logfile.1 logfile.2 logfile.3 logfile.4 logfile.5    /dev/null
