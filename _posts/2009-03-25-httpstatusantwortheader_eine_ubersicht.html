Die Status-Antwort-Header des HTTP-Protokolls in einer tabellarischen Übersicht mit kurzen Erklärungen.

Grundlagen

Der Status-Antwortheader zeigt die Version des laufenden Servers zusammen mit einem Ergebniscode und einer optionalen Meldung an.

Beispiel: 

 HTTP/1.0 200 OK
HTTP/1.0 ist die HTTP-Version, 200 der Ergebniscode und OK die Meldung, die den Ergebniscode beschreibt.

Die Ergebniscodes sind in fünf Gruppen eingeteilt:

  1. Informative Meldungen für den Client 1xx
  2. Erfolg 2xx
  3. Umleitung 3xx
  4. Client-Fehler 4xx
  5. Server-Fehler 5xx

Die erste Stelle/Ziffer des dreistelligen Ergebniscodes zeigen die Hauptgruppe an, die beiden folgenden Stellen die Details.

Informative Meldungen für den Client 1xx

100 CONTINUE Ein Teil der Anfrage wurde bereits bearbeitet. Der Client kann seine Anfrage beenden, wenn er möchte.
101 SWITCHING PROTOCOLS Der Server möchte zu dem Protokoll wechseln, das im Upgrade-Header angegeben ist.

Erfolg 2xx

200 OK Die Anforderung war erfolgreich.
201 CREATED Nach einem POST-Befehl Erfolg; der Textteil der Antwort zeigt den URI, unter dem das neu erstellte Dokument erkannt wird.
202 ACCEPTED Die Anforderung wurde zur Bearbeitung angenommen, doch wurde die Bearbeitung nicht abgeschlossen.
203 PARTIAL INFORMATION
203 NON-AUTHORITIVE INFORMATION		 Als Antwort auf einen GET-Befehl zeigt dieser Code, daß die zurückgegebene Metainformation kein endgültiges SET des Objekts vom Server ist, sondern von einem privaten, überlagertem Web kommt.
204 NO RESPONSE
204 NO CONTENT		 Der Server hat die Anforderung erhalten, doch gibt es keine Information, die zurückgeben werden kann. Der Client sollte in derselben Dokumentenansicht bleiben. Dies wird hauptsächlich zur Zulassung von Eingaben in Scripts ohne Änderung des Dokuments verwendet.
205 RESET CONTENT Der Client soll die Werte des aktuellen HTML-Formulars auf die Defaultwerte zurücksetzen.
206 PARTIAL CONTENT Ein Teil des gewünschten Objektes wird nach einer Anfrage mit Range-Header zurückgegeben.

Umleitung 3xx

300 MULTIPLE CHOICES Die angeforderte URL verweist auf mehr als ein Objekt. Weitere Informationen werden dem Entity-Body entnommen.
301 MOVED PERMANENTLY Den angeforderten Daten wurde ein neuer URI zugeordnet. Die Änderung ist von Dauer. Die Antwort enthält eine Zeile, die die alternative Adresse des Dokuments angibt.
302 MOVED TEMPORARILY Die angeforderten Daten befinden sich tatsächlich unter einem anderem URI. Die Änderung ist nicht von Dauer. Die Antwort enthält eine Headerzeile, die die alternative Adresse des Dokumentes angibt.
303 METHOD
303 SEE OTHER		 Diese Antwort teilt dem Client mit, das er eine andere Netzwerkadresse probieren soll. Eine Bestästigung des Nutzers sollte eingeholt werden.
Es kann auch eine andere Methode als GET verwendet werden. Der zurückgegebene Rumpfabschnitt (Entity-Body) enthält die zu verwendende Methode und deren Parameter.
304 NOT MODIFIED
304 USE LOCAL COPY		 Falls der Client eine bedingte GET-Anfrage, z.B 'If-Modified-Since', startet und das Dokument nicht geändert wurde, wird diese Antwort generiert.
305 USE PROXY Auf die gewünschte URL kann nur über den im Location-Header angegeben Proxy zugegriffen werden.

Client-Fehler 4xx

400 BAD REQUEST Die Anforderung hat eine falsche Syntax oder konnte nicht bedient werden.
401 UNAUTHORIZED Unzulässige Zugriffsberechtigung. Der Client sollte die Anfrage mit einem passendem (Authenticate)-Header wiederholen.
402 PAYMENT REQUIRED Ungültiges Verrechnungsschema. Für zukünftige Protokolle reserviert.
403 FORBIDDEN Die Anforderung bezieht sich auf etwas Verbotenes, da hilft auch keine Zugangsberechtigung.
404 NOT FOUND Der Server hat nichts gefunden, was mit der Anforderung übereinstimmt.
405 METHOD NOT ALLOWED Die verwendete Methode ist nicht erlaubt.
406 NOT ACCEPTABLE Bei einer Content-Negotiation wurde keine passende Variante gefunden.
407 PROXY AUTHENTICATION REQUIRED Der Proxy-Server vermisst den Authenticate-Header.
408 REQUEST TIME-OUT Die Anfrage konnte nicht innerhalb eines bestimmten Zeitlimits bearbeitet werden.
409 CONFLICT Der Server hat einen Konflikt bzzgl. einer anderen Anfrage oder seiner eigenen Konfiguration festgestellt. Weitere Informationen befinden sich im Entity-Body.
410 GONE Das angeforderte Objekt ist nicht mehr vorhanden. Es ist auch kein neuer URI bekannt.
411 LENGTH REQUIRED Der Content-Length-Header fehlt.
412 PRECONDITION FAILED Die Bedingungen, die in einem if-Header spezifiziert wurden, konnten nicht erfüllt werden.
413 REQUEST ENTITY TOO LARGE Der Server kann die Anfrage nicht bearbeiten, da der Entity-Body zu lang ist.
414 REQUEST URI TOO LARGE Der Server kann die Anfrage nicht bearbeiten, da der Request-URI zu lang ist.
415 UNSUPPORTED MEDIA TYPE Der Server kann die Anfrage nicht bearbeiten, da die Daten im Entity-Body in einem unbekannten Format vorliegen.

Server-Fehler 5xx

500 INTERNAL SERVER ERROR Der Server hat eine unerwartete Bedingung angetroffen, die ihn von der Erfüllung der Anforderung abhält. Eine genaue Fehlerangabe entfällt. :-(
Diese Meldung erscheint z.B. bei CGI-Script-Fehlern oder wenn der Server nicht genügend Ressourcen bereitstellen kann.
501 NOT IMPLEMENTED Der Server unterstützt diese Anforderung nicht.
502 BAD GATEWAY Der Server hat von einem Gateway oder einem anderen Server eine ungültige Antwort erhalten.
503 SERVICE UNAVAILABLE Der Server kann die Anfrage wegen Überlastung oder laufender Pflege nicht bedienen.
504 GATEWAY TIMEOUT Die Antwort kam nicht innerhalb der Zeit, die das Gateway gewillt war zu warten. Wie 503, jedoch mehr Diagnose-Information.
505 HTTP VERSION NOT SUPPORTED Der Server kann die Anfrage nicht bearbeiten, da die verwendete HTTP-Version nicht unterstützt wird.
506 VARIANT ALSO VARIES Content-Negotiation-Fehler. Es konnte keine gültige Variante ermittelt werden.
